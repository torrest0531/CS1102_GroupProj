	<h1>All About Links</h1>

	<h2 id="dissecting_a_url">Dissecting a URL</h2>

	<figure>
		<img src="url-parts.jpg" alt="The Parts of a URL explained" />
		<figcaption>The parts of a URL. From <em>Learning Web Design</em>, p. 24</figcaption>
	</figure>

	<ul>
		<li><strong>Protocol:</strong> The first part of a URL is the protocol. On the web, we almost always use Hypertext Transfer Protocol (HTTP) or HTTPS, which is simply a more secure version of HTTP.</li>
		<li><strong>Host Name:</strong> The hostname points to a specific web site within a domain. Usually it is www, but this is not necessarily always the case. The www is so ubiquitous that it is oftentimes left off of urls to make them more clear.</li>
		<li><strong>Domain Name:</strong> The domain name is the unique identifier of the site.</li>
		<li><strong>Absolute Path:</strong> The absolute path tells us the path to the requested file on the web server. </li>
		<li><strong>Directory Path:</strong> This part of the URL tells us what directories the requested file is contained in.</li>
		<li><strong>Document:</strong> The last part of the URL always ends in a file name, with it's extension.</li>
	</ul>

	<p>Review pages 24-25 in <em>Learning Web Design</em> for more information.</p>


	<h2>The a Element</h2>
	<p>The <code>a</code> element (anchor) is used to insert links into an HTML document. The <code>a</code> element wraps around the text you want to link be a link. You must use attributes to tell the link what to do.</p>

	<p>The <code>a</code> element has many possible attributes. Two of the most important are <code>href</code> and <code>title</code>:</p>
	<ol>
		<li><strong>href:</strong> The location (URL) of the file you are linking to</li>
		<li><p><strong>title:</strong> The title or a description of the linked document<br />
			Depending on the browser, this attribute value may show as a tooltip (the content will pop up when you hover your mouse cursor over it).</li>
	</ol>
	
	<h3>Example Link</h3>
	<p>Here in an example link to the Simmons University website.</p>
		<pre><code class="language-markup">&lt;a href=&quot;<span class="code-attribute-value">http://www.simmons.edu/</span>&quot title=&quot;<span class="code-attribute-value">Visit the Simmons University website</span>&quot;&gt;</span>Simmons University&lt;/a&gt;</code></pre>

	<h2>Types of links</h2>

	<h3 id="absolute_urls">Absolute URLs</h3>

	<p>Absolute URLs use the entire URL, including the protocol (<strong>http://</strong>), to link to a page. These are usually used to link to an external site or to your home page.</p>

	<p>An absolute link is created when the href value is a fully qualified URL, including:</p>
	<ul>
		<li>the protocol: <code>http://</code> or <code>https://</code></li>
		<li>domain name: e.g. <code>www.yourdomain.com</code></li>
		<li>and filename: e.g. <code>yourpage.html</code> (may be blank if you're requesting the index.html page)</li>
	</ul>

	<p>You must use absolute URLs when you are linking to an external site.</p>
	<hr>

	<h4 id="example:_absolute_urls_">Example: Absolute URLs </h4>
	<p>To link to the Simmons website home I would use the following:</p>

	<pre>
		<code class="language-markup">&lt;a href=&quot;<mark>http://www.simmons.edu/</mark>&quot;&gt;Simmons University&lt;/a&gt;</code></pre>

	<p>To link to the class home page from this page I would use the following:</p>

	<pre>
		<code class="language-markup">&lt;a href=&quot;<mark>http://web.simmons.edu/~grovesd/comm244</mark>&quot;&gt;Comm 244&lt;/a&gt;</code></pre>

	<p><em>Note: In both examples, we are linking to the <code>index.html</code> page and therefore, the filename can be left out.</em></p>


	<h3 id="relative_urls">Relative URLs</h3>
	<p>Relative URLs are used to link to a file <strong>relative</strong> to the file in which you are adding the link element.</p>
	
	<p><strong>This is where it becomes very important to understand the directory structure and filenames in your website!</strong></p>
	
	<p>Computer Filesystems:</p>
	<ul>
		<li>Filesystems on computers are arranged hierarchically in folders.</li>
		<li>Each file resides inside one or more nested folders.</li>
	 
		<li>Websites work exactly the same way.</li>

		<li>The top level folder of our website is referred to as the <strong>site root</strong> or simply <strong>root level</strong>.</li>
	</ul>


	<hr>

	<h3 id="relative_urls">Relative URLs (continued)</h3>
	<p>To use relative URLs, we have to be able to navigate the directory structure of our sites.</p>
	
	<p>We have to build a link by describing how to get from one html page to another.</p>

	<!-- <p>When we are working with relative URLs, we use the forward slash ( <strong>/</strong> ) to designate a directory (folder). We use two periods and then a forward slash ( <strong class="addspace">../</strong> ) to move up one directory.</p> -->
	
	We use the following symbols to describe navigating folders:

	<ul>
		<li>Go inside a folder (or down one level in the hierarchy): <mark><code> / </code></mark></li>
		<li>Go outside a folder (or down up level in the hierarchy): <mark><code> ../ </code></mark></li>
	</ul>
	
	<p>Our links will consist of two parts: Path to file + file name</p>

	<p><span class="definition">Root Level</span>: When we say root level or root directory, we are referring to the entry level directory of a website. This directory usually contains the homepage.</p>

	<h2 id="giving_our_link_titles">Giving Our Links Titles</h2>
	<p>We should always make sure to add titles to our links using the <code>title</code> attribute. The title attribute gives extra information about what we are linking to. This is especially useful for screen readers.</p>

	<p>Example: Linking to the Simmons College website</p>

	<code>
		<pre>&lt;a href=&quot;http://www.simmons.edu/&quot; <mark>title="View the Simmons College Website"</mark>&gt;Simmons College&lt;/a&gt;</pre>
	</code>
